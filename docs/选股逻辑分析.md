# A股AI选股系统 - 完整逻辑分析

## 架构设计：慢链路（选股）+ 快链路（决策）

```
盘前预热（慢）          早盘决策（快90秒）        盘后复盘
    ↓                      ↓                      ↓
生成候选池 → 存快照 → 读快照打分决策 → 执行 → 分析结果
```

---

## 一、选股维度（6大数据源）

### 1. **涨停板候选** (`_get_limit_up_candidates`)
- 数据源：东方财富涨停板实时数据
- 打分规则：
  - 基础分 40分
  - 每连板 +15分
  - 上限 80分
- 示例：3连板 = 40 + 3×15 = 85分 → 截断到80分

### 2. **龙虎榜候选** (`_get_dragon_tiger_candidates`)
- 数据源：东方财富龙虎榜数据
- 打分规则：
  - 净买额 > 0：15分 + (净买额/1000万) → 上限25分
  - 净买额 < 0：10分 + (净买额/1000万) → 下限0分

### 3. **北向资金候选** (`_get_north_flow_candidates`)
- 数据源：东方财富北向持股数据
- 打分规则：
  - 持股市值 > 500亿：25分
  - 持股市值 100-500亿：15-25分线性
  - 持股市值 10-100亿：5-15分线性
  - 流通占比 > 10%：额外+5分

### 4. **业绩预增候选** (`_get_performance_candidates`)
- 数据源：东方财富业绩预告
- 打分规则：
  - 预增幅度 ≥200%：35分
  - 预增幅度 100-200%：25-35分
  - 预增幅度 50-100%：15-25分
  - 预增幅度 20-50%：8-15分

### 5. **机构调研候选** (`_get_survey_candidates`)
- 数据源：东方财富机构调研记录
- 打分规则：
  - 调研次数 ≥20次：35分
  - 调研次数 10-20次：25-35分
  - 调研次数 5-10次：15-25分
  - 含基金/保险：额外+5分

### 6. **异动放量候选** (`_get_volume_candidates`)
- 数据源：东方财富实时行情推送
- 过滤条件：量比>3 且 涨跌幅>-5%
- 打分规则：
  - 量比 ≥10：25分
  - 量比 5-10：15-25分
  - 量比 3-5：8-15分
  - 涨幅2-7%：+5分
  - 换手率5-15%：+3分

## 二、候选池生成流程

### 多源叠加奖励机制
```python
# stock_picker.py _merge_candidates()
单源候选：各自维度得分
双源候选：两个维度得分相加 + 10分奖励
三源候选：三个维度得分相加 + 20分奖励
四源候选：+30分
五源候选：+40分
六源全中：+50分（理论上限）

最终得分上限：100分
```

### 过滤规则
```python
# stock_picker.py _is_valid()
✗ ST股/退市股（名称含"退"）
✗ 北交所（代码8/4开头）
✗ 科创板（代码688开头）
✗ 名称过短（<2字符）
✗ 得分 < min_score（默认55分）
```

## 三、不同时段的选股策略

### 场景1：盘前预热（周末/收盘后/早盘前）
**时间**：非交易时段  
**调用**：`python main.py --prefetch` 或 Hermes `quant_prefetch.sh`  
**流程**：
```python
# scheduler/pipeline.py prefetch()
1. 调用 pick_stocks() 生成完整候选池（6维度全扫）
2. 拉取候选股K线、基本面、新闻
3. 批量舆情分析（LLM）
4. 存入快照：candidate_pool_YYYYMMDD.json
```

**选股方法**：`pick_stocks(top_n=10)`
- 6维度全开：涨停板+龙虎榜+北向+业绩+调研+放量
- 返回Top 10

---

### 场景2：早盘扫描（9:35）
**时间**：开盘后5分钟  
**调用**：`python main.py --scan` 或 Hermes `quant_scan.sh`  
**预算**：90秒硬超时  
**流程**：
```python
# scheduler/pipeline.py fast_scan()
0-10s:  读昨日候选池快照（如果有）
        如果无快照 → 华泰选股（pick_stocks_by_htsc，30秒超时）
10-30s: 读市场环境快照 + 更新实时价格
30-60s: 并发打分（5维信号）
60-75s: LLM决策TopK
75-90s: 输出TradePlan
```

**选股方法**：
- **优先**：读快照（盘前预热的结果）
- **降级**：`pick_stocks_by_htsc(max_candidates=10)` - 华泰自然语言选股
  - 默认条件："主力净流入超过3000万且换手率大于3%"
  - 30秒超时，失败返回空列表

---

### 场景3：盘中扫描（11:28/14:00）
**时间**：盘中关键时点  
**调用**：Hermes `quant_midday_scan.sh`  
**流程**：同早盘扫描（复用`fast_scan()`）  
**差异**：
- 候选池快照可能已过期（4小时有效期）
- 市场快照更新（涨停板数量实时变化）

---

### 场景4：周末选股（无交易日）
**时间**：周末/节假日  
**调用**：手动 `python main.py --scan`  
**流程**：
```python
# 无盘前预热快照，直接选股
1. pick_stocks() 6维度扫描（慢，可能2-5分钟）
2. 或 pick_stocks_by_strategy() 技术形态扫描
3. 或 pick_stocks_by_htsc() 华泰条件选股
```

**典型用法**：
```python
# 技术形态选股（不依赖实时数据）
pick_stocks_by_strategy(
    strategy_names=["zt_reversal", "ma_cross"],
    max_candidates=200
)
```

## 四、快慢链路对比

| 维度 | 慢链路（盘前预热） | 快链路（盘中扫描） |
|------|-------------------|-------------------|
| 时间 | 不限（2-10分钟） | 90秒硬超时 |
| 选股方法 | 6维度全扫 | 读快照/华泰选股 |
| 候选数量 | Top 10-30 | Top 10 |
| LLM调用 | 批量舆情分析 | 只对TopK决策 |
| 数据存储 | 存快照供盘中读 | 读快照，不存 |
| 失败降级 | 无（重试） | 跳过该步骤 |

## 六、核心问题与不足

### 问题1：早盘依赖昨日候选池
```python
# 如果盘前未执行prefetch，早盘会降级到华泰选股
# 华泰选股只返回10只，且有30秒超时风险
```

**影响**：
- 周末→周一早盘：昨日快照48小时过期，必走华泰选股
- 华泰选股失败 → 无候选 → 整个扫描失败

### 问题2：选股池更新频率低
```python
# 候选池快照有效期4小时，盘中不更新
# 涨停板10:30打开 → 候选池仍是9:25的数据
```

### 问题3：周末选股无法利用实时数据
- 涨停板/放量/龙虎榜都依赖交易日数据
- 周末只能用技术形态选股（滞后）

### 问题4：6维度数据源单点故障
```python
# 任意API超时/失败 → 该维度候选为空
# 无降级方案（应该有备用数据源）
```

### 问题5：选股逻辑固化
- 打分权重写死（连板+15分/次）
- 无历史回溯验证（龙虎榜选出的股票实际胜率多少？）
- 无自适应调整（某维度连续失效也不降权）

## 总结

**当前选股逻辑本质**：
- **盘前**：东财6维度数据爬取 → 打分合并 → 存快照
- **盘中**：读快照 → 5维信号打分 → LLM决策TopK
- **周末**：技术形态扫描（K线模式识别）

**最大痛点**：
1. 候选池静态（盘中不更新）
2. 华泰选股是唯一实时降级，且不稳定
3. 选股规则固化，无自我优化

**事件驱动改进方向**：
- 涨停打开实时通知 → 动态加入候选池
- 异动放量实时触发 → 立即扫描该股
- 龙虎榜实时推送 → 盘中更新候选池
