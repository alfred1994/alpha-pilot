# Quant Pilot 盘中盯盘优化交接

日期：2026-06-22

本文给 Hermes agent 做后续接手参考。当前代码已在本地工作区实现，并已同步到服务器 `/home/ubuntu/projects/quant-pilot`。

## 本次目标

把 `--auto` 从固定定时扫描改成更接近真人交易员的盘中盯盘模式：

- 每分钟仍做持仓止损巡检。
- 每 5 分钟做轻量看盘，记录“看过盘”和不交易原因。
- 早盘没有选出股票时，盘中继续观察，不再沉默一整天。
- 高现金/低仓位且市场转强时生成疑似踏空事件。
- 观察池候选必须连续两次确认，才允许救援扫描。
- 救援扫描只看 watchlist Top 候选，且买入仓位缩小到普通仓位的 40%。
- LLM `HOLD`、超时、解析失败时不再兜底转 `BUY`。
- 14:30 后不新增追入机会，只做持仓和卖出管理。

## 关键代码

- `config.py`
  - 新增默认配置：
    - `AUTO_WATCH_INTERVAL=300`
    - `AUTO_RESCUE_SCAN_INTERVAL=900`
    - `AUTO_WATCHLIST_TTL=3600`
    - `AUTO_RESCUE_MAX_TOPK=5`
    - `AUTO_RESCUE_POSITION_SCALE=0.4`

- `scheduler/intraday_watch.py`
  - 新增盘中轻量盯盘模块。
  - 维护 `data/intraday_watchlist.json`。
  - `run_watch_cycle(...)` 负责轻量看盘、早盘基准、踏空检测、watchlist 更新、二次确认。
  - `filter_trade_plan_for_rescue(...)` 只允许确认白名单里的 `BUY`，并缩小仓位；`SELL` 保留。

- `scheduler/auto_trader.py`
  - `AutoTraderState` 新增：
    - `last_watch_at`
    - `last_rescue_scan_at`
    - `morning_baseline`
    - `watchlist_count`
    - `missed_opportunity_count`
    - `rescue_scan_count`
  - 盘中循环新增轻量看盘和救援扫描。
  - 写入 `auto_events`：
    - `watch_cycle`
    - `watchlist_update`
    - `missed_opportunity`
    - `rescue_scan`
    - 原有 `auto_cycle`
  - 救援扫描后会刷新 `last_scan_at` 和 `last_execute_at`，避免下一轮立刻跑未过滤全量扫描。

- `scheduler/pipeline.py`
  - `fast_scan/run_scan` 支持 `candidate_codes` 和 `candidate_items`，用于救援扫描白名单。
  - 救援扫描不会再触发可转债 T+0 旁路扫描。
  - 删除 LLM 全部 `HOLD` 后保留最高分候选的兜底逻辑。
  - 未获得 LLM `BUY/SELL` 明确动作时写入 `HOLD_NO_BUY_DECISION`，不生成买单。
  - 买入未成交不再计入 `executed_orders`。

- `strategy/llm_trader.py`
  - `make_decision` 支持 `llm_retries` 和 `llm_timeout`。
  - 快链路用短超时和不重试，避免盘中被 LLM 卡死。

- `review/ai_trader_report.py`、`scheduler/ops_status.py`
  - 增加盘中观察统计：
    - 轻量看盘次数
    - 疑似踏空次数
    - 救援扫描次数
  - 报告统计自动循环时只把 `event_type=auto_cycle` 算作循环，避免新增观察事件放大循环数。

- `data/linux_tasks/run_doctor.sh`、`data/linux_tasks/systemd_user/quant-pilot-doctor.service`、`scheduler/linux_tasks.py`
  - Doctor 增加超时保护。
  - 脚本层：`timeout --kill-after=15s 240s`
  - systemd 层：`TimeoutStartSec=300`

- `.gitattributes`
  - 固定 `.sh`、`.service`、`.timer` 使用 LF，避免 Windows 同步到 Linux 后 bash 读到 CRLF。

## 重要行为约束

- 当前仍强制模拟盘：`BROKER_MODE=paper`。
- 救援扫描不是直接追入，必须先进入 watchlist，连续两次确认后才允许进 LLM。
- 救援买入只允许确认白名单中的 `BUY`。
- LLM `HOLD`、无响应、解析失败，都不会生成 `BUY`。
- 14:30 后不新增追入机会。
- 如果 watchlist 标的确认后本轮消失，会降级为 `cooling`，不会进入救援白名单。
