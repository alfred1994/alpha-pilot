# 选股优化说明

## 问题诊断

原有选股系统主要从以下维度选股，容易选到**高位股**：

1. **涨停板** - 连板数越多分越高（已经大涨）
2. **龙虎榜** - 净买入大的通常已经拉升过
3. **异动放量** - 量比>3，可能是高位放量
4. **资金流向** - 主力净流入，往往是追涨资金

这些维度都是"追强势股"的逻辑，适合短线博弈，但容易买在高位。

## 优化方案

新增**低位潜力股选股器**（`strategy/low_position_picker.py`），专注挖掘：

### 1. 回撤反弹股
- **逻辑**：曾经涨停（有热度），但现在回落（充分回调）
- **数据源**：过去5天涨停板历史
- **评分**：越早的涨停分数越高（回调越充分）

### 2. 低价股
- **逻辑**：股价5-15元，避免高价股风险
- **数据源**：活跃股实时行情
- **评分**：价格越低分越高，小幅上涨加分

### 3. 稳健型
- **逻辑**：龙虎榜净买适中（非爆炒），涨幅不大（未追高）
- **数据源**：龙虎榜数据
- **评分**：净买额1000-5000万，涨幅<5%

## 使用方式

### 方式1: 纯低位模式（推荐新手）
关闭追高维度，只选低位股：

```python
from strategy.stock_picker import pick_stocks

candidates = pick_stocks(
    top_n=15,
    min_score=30,
    low_position_mode=True,  # 自动关闭涨停板/龙虎榜/异动放量
)
```

### 方式2: 混合模式（高手使用）
低位和追高都保留，让分数说话：

```python
candidates = pick_stocks(
    top_n=20,
    min_score=40,
    use_low_position=True,  # 加入低位维度
    low_position_mode=False,  # 保留其他维度
)
```

### 方式3: 独立使用低位选股器

```python
from strategy.low_position_picker import pick_low_position_stocks

candidates = pick_low_position_stocks(
    top_n=20,
    min_score=25,
)
```

## 命令行使用

修改 `main.py` 的选股调用：

```python
# 纯低位模式
candidates = pick_stocks(low_position_mode=True)

# 或混合模式
candidates = pick_stocks(use_low_position=True)
```

然后运行：

```bash
python main.py --scan
```

## 评分逻辑对比

### 原有维度（追高）
- 涨停板：连板越多分越高（40-80分）
- 龙虎榜：净买额越大分越高（15-25分）
- 异动放量：量比越高分越高（8-25分）

### 新增维度（低位）
- 回撤反弹：回调天数越多分越高（25-35分）
- 低价股：价格越低分越高（15-25分）
- 稳健型：适中指标加分（20-25分）

### 多源叠加奖励
- 2个来源：+10分
- 3个来源：+20分
- 4个来源：+30分
- 5个来源：+40分

## 注意事项

1. **低位≠价值投资** - 这里的"低位"是技术面概念（回调充分），不是基本面估值
2. **仍需风控** - 低位股也可能继续下跌，止损仍然重要
3. **数据源限制** - 某些API可能不稳定，已改用东方财富历史数据作为替代
4. **适合中长线** - 低位布局需要等待反弹，不适合超短线

## 后续优化方向

1. **增加基本面筛选** - PE/PB/ROE等估值指标（API稳定后）
2. **技术形态识别** - MACD底背离、RSI超卖等
3. **行业轮动** - 识别低位但行业向上的板块
4. **解禁预警** - 过滤30天内有重大解禁的股票
